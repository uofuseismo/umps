◆ setReceiveTimeOut()

void UMPS::Services::Command::RequestorOptions::setReceiveTimeOut ( const std::chrono::milliseconds &  timeOut)

Sets the time out for receiving replies from the local module.

Parameters
[in]timeOutThe receive time out. If this is negative then this program will hang indefinitely until the program you are interacting with responds - this is problematic if your thread has other things to do. If this zero then this requestor will return immediately and probably miss the response. In general, it's a good idea to wait a few hundredths of second for a response.
