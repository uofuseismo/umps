Detailed Description

Options for initializing the publisher in the pick broadcast.

The publisher sends picks and it is up to the subscribers to be listening.

void UMPS::ProxyBroadcasts::Pick::PublisherOptions::setTimeOut ( const std::chrono::milliseconds &  timeOut)
noexcept

If the publisher waits the timeOut length of time before sending a message then it will without having sent the message. This is useful when the publisher thread has other responsibilities.

Parameters
[in]timeOutThe time out duration in milliseconds. If this is zero then the subscriber will immediately return. If this is negative then the subscriber will wait indefinitely until a message is received.
