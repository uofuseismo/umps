◆ setSocketOptions()

void UMPS::Authentication::ZAPOptions::setSocketOptions ( zmq::socket_t *  socket) const

A utility routine that sets the ZAP options on the ZeroMQ socket.

Parameters
[in,out]socketThe ZeroMQ socket. On input, this socket must not be connected when this is called, i.e., use this method before a zmq_bind or zmq_connect call. On exit, the appropriate ZeroMQ ZAP options for the desired security level will be set on the socket.
Exceptions
std::runtime_errorif the options cannot be set.

◆ setStonehouseClient()

void UMPS::Authentication::ZAPOptions::setStonehouseClient ( const Certificate::Keys &  serverKeys,
const Certificate::Keys &  clientKeys 
)

This enables the stonehouse security pattern for the client. The client must know the server's public key. Additionally, the client must have a valid public and private key pair. The public key will be encrypted, sent to the server, and validated. Additionally, the IP may be validated.

Parameters
[in]serverKeysThe server's public key.
[in]clientKeysThe client's public and private key pair.
Exceptions
std::invalid_argumentif server.havePublicKey() is false, client.havePublicKey() is false, or client.havePrivateKey() is false.
Note
Upon successful completion, isAuthenticationServer() will be false.

This enbales the stonehouse security pattern for the server. The server will be provided the client's public key and IP on getDomain() then an auxiliary service will verify.

Exceptions

◆ setWoodhouseClient()

void UMPS::Authentication::ZAPOptions::setWoodhouseClient ( const Certificate::UserNameAndPassword &  credentials)

This enables the woodhouse security pattern for the client. Here, the client provides their username and password which ZeroMQ will pass to the server during authentication. Additionally, the IP may be checked.

Parameters

Exceptions
