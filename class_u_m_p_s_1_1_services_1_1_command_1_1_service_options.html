◆ setPollTimeOut()

void UMPS::Services::Command::ServiceOptions::setPollTimeOut ( const std::chrono::milliseconds &  timeOutInMilliSeconds)
noexcept

When the service starts a thread will monitor messages on the endpoint by "polling." This controls how long the thread will wait.

Parameters
[in]timeOutInMilliSecondsThe timeout in milliseconds. If this is negative then the thread will wait forever for a message (dangerous). If this is 0 then thread will return immediately (this is a good way to get a thread to report as 100% in use in a system monitor).
