Detailed Description

Defines the reply socket options.

After receiving a message from the dealer the router will use the callback function defined in this class to reply to the request. It is important that the callback not throw or fail.

◆ setCallback()

void UMPS::Messaging::RouterDealer::ReplyOptions::setCallback ( const std::function< std::unique_ptr< UMPS::MessageFormats::IMessage >(const std::string &messageType, const void *data, size_t length)> &  callback)

The callback function is specified by the user to process messages.

Your application will create a specific request/reply pair. The router/dealer will forward request messages to the reply service. Nominally, the callback will validate the message type and, based on the type, appropriately convert the message payload to an appropriate request IMessage. The request will then be processed and responded to with a corresponding response IMessage that the requestor can handle.

Parameters
[in]callbackThe callback function which processes the message.
Note
It is important this function never throw, crash, etc. This is when exiting gracefully matters.
