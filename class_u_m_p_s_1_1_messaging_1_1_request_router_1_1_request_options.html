◆ setTimeOut()

void UMPS::Messaging::RequestRouter::RequestOptions::setTimeOut ( const std::chrono::milliseconds &  timeOut)
noexcept

If the requests waits the timeOut length of time before receiving a response then it will return without a message. This is useful when the requestor thread has other responsibilities.

Parameters
[in]timeOutThe time out duration in milliseconds. If this is zero then the requestor will immediately return. If this is negative then the requestor will wait indefinitely until a message is received.
